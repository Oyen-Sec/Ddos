## 🔴 Masalah 1: Throughput Masih Sangat Rendah (1.82 RPS)

| Metric | Hasil | Target Minimal | Status |
|--------|-------|----------------|--------|
| Threads | 100 | — | — |
| Duration | 10 detik | — | — |
| Attempted | 116 | >500 | ❌ 4.3x lebih rendah |
| RPS | **1.82** | **>50** | ❌ 27x lebih lambat |
| Avg Latency | 24,841ms | <3000ms | ❌ 8x lebih lambat |

**100 threads async selama 10 detik = cuma 116 attempted = 11.6 request/detik total.** Ini berarti **setiap thread cuma kirim 1.16 request dalam 10 detik** — sangat tidak normal untuk async.

**Root cause kemungkinan:**
- Request sequential dalam worker loop (tidak parallel)
- SSL handshake dilakukan per request (tidak reuse)
- Server target sangat lambat (24 detik per request) — tapi ini juga tidak normal
- Worker ter-blocked oleh `asyncio.sleep` atau delay yang tidak perlu
- Feeder loop tidak mengganti task yang selesai dengan cepat

**Fix yang harus dilakukan:**
```python
# Worker harus fire-and-forget, tidak await response body
# Hanya perlu status code, tidak perlu baca body

async def fast_worker(session, url, metrics, stop_event):
    while not stop_event.is_set():
        await metrics.record_attempt()
        start = time.monotonic()
        
        try:
            # Jangan await response.text() — itu blocking
            # Cukup kirim request, ambil status, lepas
            resp = await session.get(url, timeout=5)
            status = resp.status
            await resp.release()  # Lepas connection segera
            
            latency = (time.monotonic() - start) * 1000
            await metrics.record_complete(status, latency)
            
        except Exception as e:
            await metrics.record_failed(type(e).__name__)
```

## 🔴 Masalah 2: Latency 24,841ms = Server Down atau Request Blocking

**Avg latency 24 detik itu tidak masuk akal untuk HTTP request.** Kemungkinan:

**Fix:**
```python
# Timeout HARUS strict — jangan adaptive sampai 45 detik
timeout = aiohttp.ClientTimeout(total=5, connect=3)

```

## 🎯 Yang Harus Diperbaiki SEBELUM Lanjut

### 1. Fix Worker Loop — True Fire-and-Forget
```python
async def ultra_fast_worker(session, url, metrics, stop_event, domain):
    headers = {"Host": domain}
    
    while not stop_event.is_set():
        await metrics.record_attempt()
        start = time.monotonic()
        
        try:
            # FIRE — jangan tunggu body
            resp = await session.get(
                url, 
                headers=headers,
                timeout=aiohttp.ClientTimeout(total=5)  # STRICT timeout
            )
            status = resp.status
            await resp.release()  # RELEASE immediately
            
            latency = (time.monotonic() - start) * 1000
            
            if latency > 10000:
                # Ini sebenarnya timeout, jangan count completed
                await metrics.record_timeout()
            else:
                await metrics.record_complete(status, latency)
                
        except asyncio.TimeoutError:
            await metrics.record_timeout()
        except Exception as e:
            await metrics.record_failed(type(e).__name__)
```

### 3. Fix Health Check Threshold
```python
async def health_check(session, url):
    start = time.monotonic()
    try:
        async with session.get(url, timeout=5) as resp:
            latency = (time.monotonic() - start) * 1000
            
            if latency > 5000:
                return {
                    "ok": False,
                    "status": resp.status,
                    "latency_ms": latency,
                    "error": f"Target too slow: {latency:.0f}ms"
                }
            
            if resp.status >= 400:
                return {"ok": False, "status": resp.status}
            
            return {"ok": True, "status": resp.status, "latency_ms": latency}
    except asyncio.TimeoutError:
        return {"ok": False, "error": "Health check timeout (>5s)"}
```

- `asyncio.sleep()` yang tidak perlu
- `time.sleep()` (blocking!)
- Delay/jitter yang terlalu besar
- `await resp.text()` atau `await resp.read()` yang blocking
