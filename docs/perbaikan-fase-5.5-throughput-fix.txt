ROOT CAUSE ANALYSIS
-----------------------------------------------------------

BUG 1: Double Release Response
File: src/core/ultra_worker.py (Line 18-19)
```python
async with session.get(...) as resp:
    status = resp.status
    await resp.release()  # ❌ WRONG: Context manager auto-releases
```

Masalah:
- Context manager `async with` OTOMATIS release response ketika exit
- `await resp.release()` DALAM context manager = double release
- Double release = connection corruption / pool deadlock
- Workers ter-block menunggu connection pool yang deadlock

FIX:
- Remove `await resp.release()` explicit
- Let context manager handle auto-release
- Add `allow_redirects=False` untuk faster response

BUG 2: TCPConnector Pool Terlalu Kecil
File: src/core/universal_attack.py (Line 56-60)
```python
connector = aiohttp.TCPConnector(
    limit=threads * 2,          # 200 untuk 100 threads
    limit_per_host=threads,     # 100 per host
    ...
)
```

Masalah:
- 100 workers = 100 concurrent requests
- limit=200 total, limit_per_host=100
- limit_per_host=100 = BOTTLENECK
- Hanya 100 requests bisa concurrent, worker #101+ ter-block di pool queue
- Untuk 100 threads + overhead = butuh 500+ connections

FIX:
```python
connector = aiohttp.TCPConnector(
    limit=threads * 5,           # 500 total
    limit_per_host=threads * 4,  # 400 per host
    ttl_dns_cache=0,            # No DNS caching
    force_close=False,          # Keep-alive
    keepalive_timeout=30        # 30s keepalive
)
```

BUG 3: Session Timeout Terlalu Lama
File: src/core/universal_attack.py (Line 65)
```python
timeout=aiohttp.ClientTimeout(total=10, connect=5)
```

Masalah:
- total=10 detik = worker mau menunggu 10 detik sebelum timeout
- Real latency actual ≠ timeout — request bisa "hang" 10 detik
- Untuk 100 workers, 10s timeout = devastating untuk throughput
- Workers ter-block di "waiting for response" bukan "sending request"

FIX:
```python
timeout=aiohttp.ClientTimeout(total=5, connect=2, sock_read=3)
```

BUG 4: Health Check & Benchmark Timeout Tidak Konsisten
File: src/core/universal_attack.py (Line 24-25, 46)
```python
health: timeout=5  (aiohttp.ClientTimeout(total=5))
benchmark: timeout=5 (no ClientTimeout object)
workers: timeout=5 (strict_timeout in ultra_worker)
```

Masalah:
- Health check dengan timeout 5s, tapi latency >5s dianggap OK
- Tidak consistent dengan worker timeout
- Hasil health check tidak akurat

FIX:
- Health check: total=3, connect=1, sock_read=2
- Benchmark: same as workers
- Threshold: latency >3000ms = FAIL
