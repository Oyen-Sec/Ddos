============================================================
PHASE 1: RECONNAISSANCE & INTELLIGENCE GATHERING
============================================================

Objective: Discover origin IP, infrastructure, and target vulnerabilities.

1.1 DNS ENUMERATION
    - Query all record types: A, AAAA, MX, NS, TXT, SOA, CNAME
    - Zone transfer attempt (AXFR)
    - DNSSEC validation check
    - CDN detection: Cloudflare, Akamai, Fastly, AWS CloudFront
    - Output: dns_recon.json

1.2 CERTIFICATE TRANSPARENCY LOGS
    - Scan crt.sh, Censys, CertSpotter
    - Extract all SAN (Subject Alternative Names)
    - Find exposed subdomains & IPs in certificates
    - Output: cert_recon.json

1.3 HISTORICAL DNS RECORDS
    - Query SecurityTrails, ViewDNS, DNSDumpster, PassiveTotal
    - Find historical IPs before CDN implementation
    - Output: historical_dns.json

1.4 SHODAN / CENSYS / ZOOMEYE SCAN
    - Query: ssl:"target.com" OR hostname:"target.com"
    - Extract: open ports, services, banners, vulnerabilities
    - Output: shodan_recon.json

1.5 SUBDOMAIN ENUMERATION
    - Wordlist: 10M+ subdomains (subdomain.txt)
    - Techniques: brute force, permutation, certificate transparency
    - Tools: amass, subfinder, assetfinder, knockpy
    - Output: subdomain_recon.json

1.6 ORIGIN IP DISCOVERY
    - Favicon hash matching (Shodan)
    - Email header analysis
    - GitHub / Pastebin leak search
    - CloudFlair tool automation
    - Output: origin_ip.json

1.7 WAF/CDN FINGERPRINTING
    - Identify WAF vendor: Cloudflare, AWS WAF, Akamai, Imperva
    - Detect WAF rules & bypass feasibility
    - Test response to malformed requests
    - Output: waf_fingerprint.json

1.8 TECHNOLOGY STACK IDENTIFICATION
    - Wappalyzer-style detection
    - Server software, framework, CMS, database
    - JavaScript libraries & versions
    - Output: tech_stack.json

FASE 4: ATTACK VECTORS — LAYER 7 (APPLICATION)
============================================================

4.1 HTTP GET FLOOD
    - Request ke homepage & random pages
    - Random query parameters untuk cache bypass
    - User-Agent rotation
    - Header randomization
    - Rate: 10K-100K req/sec per node

4.2 HTTP POST FLOOD
    - POST ke form login, register, search
    - Payload: Random data atau valid form data
    - Target: Database overload, form processing exhaustion

4.3 SLOWLORIS
    - Partial HTTP requests
    - Keep connection open dengan slow headers
    - Target: Apache, IIS, nginx (jika config lemah)
    - Connection per target: 1000-5000

4.4 R-U-DEAD-YET (RUDY)
    - Slow POST body upload
    - Content-Length besar tapi upload lambat
    - Target: Form processing, file upload

4.5 SLOW READ
    - Request normal tapi baca response sangat lambat
    - Window size manipulation
    - Target: Server thread pool exhaustion

4.6 DYNAMIC REQUEST FLOOD
    - Random URL generation
    - Heavy query parameters (nested JSON, large arrays)
    - Target: Dynamic page generation, database query overload
    - Contoh: /search?q=[random_10KB_json]

4.7 GRAPHQL ABUSE
    - Nested query dengan depth 10+
    - Introspection query abuse
    - Alias abuse untuk resource exhaustion
    - Batch request dengan 100+ queries

4.8 WEBSOCKET FLOOD
    - Open 1000+ WebSocket connections
    - Send random binary/text frames
    - Ping flood dengan payload besar

4.9 API ENDPOINT ABUSE
    - Rate limit bypass via distributed attack
    - Expensive endpoint targeting (/export, /report, /generate)
    - Authentication endpoint brute force
